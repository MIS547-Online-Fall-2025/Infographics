🚀 What are SSH Keys?
SSH keys provide a secure way to authenticate with remote servers without passwords. They use public-key cryptography - you keep a private key secret and share the public key with servers you want to access.

Creating SSH Keys by Platform

🪟
Windows
1 Open PowerShell or Command Prompt
ssh-keygen -t ed25519 -C "your_email@example.com"
Generate a new SSH key using Ed25519 algorithm (recommended)
2 Choose Key Location
Enter file in which to save the key: (Press Enter for default)
Default location: C:\Users\YourName\.ssh\id_ed25519
3 Set Passphrase (Optional)
Enter passphrase (empty for no passphrase):
Add extra security with a passphrase (recommended)
4 Start SSH Agent
Get-Service -Name ssh-agent | Set-Service -StartupType Manual
Start-Service ssh-agent
Enable and start the SSH agent service
5 Add Key to SSH Agent
ssh-add C:\Users\YourName\.ssh\id_ed25519
Add your private key to the SSH agent
6 Copy Public Key
Get-Content C:\Users\YourName\.ssh\id_ed25519.pub | Set-Clipboard
Copy the public key to clipboard for adding to services
🍎
macOS
1 Open Terminal
ssh-keygen -t ed25519 -C "your_email@example.com"
Generate a new SSH key using Ed25519 algorithm
2 Choose Key Location
Enter file in which to save the key: (Press Enter for default)
Default location: /Users/yourusername/.ssh/id_ed25519
3 Set Passphrase (Optional)
Enter passphrase (empty for no passphrase):
Add extra security with a passphrase (recommended)
4 Start SSH Agent
eval "$(ssh-agent -s)"
Start the SSH agent in the background
5 Configure SSH Config (macOS 10.12.2+)
touch ~/.ssh/config
echo "Host *\n AddKeysToAgent yes\n UseKeychain yes\n IdentityFile ~/.ssh/id_ed25519" >> ~/.ssh/config
Configure SSH to use keychain and auto-add keys
6 Add Key to SSH Agent & Keychain
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
Add private key to SSH agent and store passphrase in keychain
7 Copy Public Key
pbcopy < ~/.ssh/id_ed25519.pub
Copy the public key to clipboard

Essential SSH Commands

🔑
Key Generation
ssh-keygen -t ed25519 -C "email@example.com"
Generate Ed25519 key (recommended)
ssh-keygen -t rsa -b 4096 -C "email@example.com"
Generate RSA key (legacy compatibility)
ssh-keygen -t ed25519 -f ~/.ssh/custom_key
Generate key with custom filename
ssh-keygen -y -f ~/.ssh/id_ed25519
Extract public key from private key
🔧
SSH Agent Management
ssh-add ~/.ssh/id_ed25519
Add key to SSH agent
ssh-add -l
List keys in SSH agent
ssh-add -d ~/.ssh/id_ed25519
Remove specific key from agent
ssh-add -D
Remove all keys from agent
🔍
Key Information
ssh-keygen -l -f ~/.ssh/id_ed25519.pub
Show key fingerprint
ssh-keygen -lf ~/.ssh/id_ed25519.pub -E sha256
Show SHA256 fingerprint
ssh-keygen -lf ~/.ssh/id_ed25519.pub -E md5
Show MD5 fingerprint
ls -la ~/.ssh/
List all SSH keys
🌐
Connection & Testing
ssh -T git@github.com
Test GitHub SSH connection
ssh -T git@gitlab.com
Test GitLab SSH connection
ssh -i ~/.ssh/custom_key user@server.com
Connect using specific key
ssh -v user@server.com
Connect with verbose output
📋
Copy Commands
cat ~/.ssh/id_ed25519.pub
Display public key (Linux/macOS)
pbcopy < ~/.ssh/id_ed25519.pub
Copy to clipboard (macOS)
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
Copy to clipboard (Windows PowerShell)
type %USERPROFILE%\.ssh\id_ed25519.pub

🔄
Key Management
ssh-keygen -p -f ~/.ssh/id_ed25519
Change key passphrase
chmod 600 ~/.ssh/id_ed25519
Set correct permissions for private key
chmod 644 ~/.ssh/id_ed25519.pub
Set correct permissions for public key

Copy public key to remote server

🔒 Security Best Practices

Use Ed25519 Keys
Ed25519 provides better security and performance than RSA. Use it unless you need legacy compatibility.
Always Use Passphrases
Protect your private key with a strong passphrase. Even if stolen, the key is useless without it.
Protect Private Keys
Never share private keys. Keep them on your local machine with proper file permissions (600).
Regular Key Rotation
Generate new keys periodically and remove old ones from servers and services.
Use SSH Agent
SSH agent securely stores decrypted keys in memory, so you don't type passphrases repeatedly.
Backup Your Keys
Keep secure backups of your private keys. Loss means losing access to all connected services.

💡 Pro Tips & Common Use Cases

GitHub/GitLab Setup
Add your public key to GitHub/GitLab SSH settings to push/pull repositories without passwords
Multiple Keys for Different Services
Create separate keys for different services (work, personal, servers) for better security isolation
SSH Config File
Use ~/.ssh/config to define aliases and default settings for different hosts and keys
Test Before Committing
Always test SSH connections with ssh -T before using them for actual work
WSL on Windows
If using WSL, you can share SSH keys between Windows and Linux or keep them separate
Key Fingerprint Verification
Always verify server fingerprints when connecting to new hosts to prevent man-in-the-middle attacks
Automating with CI/CD
Use SSH keys (without passphrases) for automated deployments and CI/CD pipelines
Emergency Access
Keep a backup authentication method in case SSH keys are lost or corrupted
