🚀 What is Terraform?
Terraform is an open-source Infrastructure as Code (IaC) tool that lets you build, change, and version infrastructure safely and efficiently. It uses a declarative language to describe your infrastructure and can manage resources across multiple cloud providers.

Terraform Workflow

1
Initialize
terraform init
Download providers and initialize working directory
2
Plan
terraform plan
Preview changes before applying them
3
Apply
terraform apply
Create or update infrastructure resources
4
Validate
terraform validate
Check configuration syntax and consistency
5
Show State
terraform show
Display current state and resources
6
Destroy
terraform destroy
Remove all managed infrastructure

Essential Terraform Commands

🚀
Core Commands
terraform init
Initialize Terraform working directory
terraform plan
Create execution plan
terraform apply
Apply configuration changes
terraform destroy
Destroy managed infrastructure
terraform validate
Validate configuration files
📋
Planning & Output
terraform plan -out=plan.tfplan
Save plan to file
terraform apply plan.tfplan
Apply saved plan
terraform plan -target=resource
Plan for specific resource
terraform output
Show output values
terraform output -json
Show outputs in JSON format
🗃️
State Management
terraform state list
List resources in state
terraform state show <resource>
Show detailed resource info
terraform state mv <src> <dest>
Move resource in state
terraform state rm <resource>
Remove resource from state
terraform refresh
Update state from real infrastructure
📥
Import & Format
terraform import <resource> <id>
Import existing infrastructure
terraform fmt
Format configuration files
terraform fmt -recursive
Format all files in subdirectories
terraform taint <resource>
Mark resource for recreation
terraform untaint <resource>
Remove taint from resource
🔧
Workspace & Debug
terraform workspace list
List available workspaces
terraform workspace new <name>
Create new workspace
terraform workspace select <name>
Switch to workspace
terraform console
Interactive console for expressions
terraform graph
Generate dependency graph
📦
Providers & Modules
terraform providers
Show provider requirements
terraform providers lock
Lock provider versions
terraform get
Download and install modules
terraform init -upgrade
Upgrade providers and modules

📄 Sample Terraform Configuration

# Configure the AWS Provider
terraform {
required_providers = {
aws = {
source = "hashicorp/aws"
version = "~> 5.0"
}
}
}
# Configure AWS Provider
provider "aws" {
region = "us-west-2"
}
# Create a VPC
resource "aws_vpc" "main" {
cidr_block = "10.0.0.0/16"
enable_dns_hostnames = true
tags = {
Name = "main-vpc"
Environment = "production"
}
}
# Create an EC2 instance
resource "aws_instance" "web" {
ami = "ami-0c94855ba95b798c7"
instance_type = "t3.micro"
subnet_id = aws_subnet.public.id
tags = {
Name = "web-server"
}
}
# Output the instance IP
output "instance_ip" {
value = aws_instance.web.public_ip
}

💡 Terraform Best Practices

Use Version Control
Always store your Terraform configurations in version control like Git
Remote State Storage
Use remote state backends (S3, Terraform Cloud) for team collaboration
State Locking
Enable state locking to prevent concurrent modifications
Use Variables
Parameterize configurations with variables for reusability
Plan Before Apply
Always run 'terraform plan' before applying changes
Use Modules
