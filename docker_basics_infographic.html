Core Concepts

üì¶
Container
A lightweight, portable package that includes everything needed to run an application: code, runtime, system tools, libraries, and settings.
Like a shipping container for your app
üéØ
Image
A read-only template used to create containers. Images are built from a series of layers and can be shared via registries like Docker Hub.
Blueprint for creating containers
üìÑ
Dockerfile
A text file containing instructions to build a Docker image. Each instruction creates a new layer in the image.
Recipe for building images
üè™
Registry
A service for storing and distributing Docker images. Docker Hub is the default public registry, but private registries are also common.
App store for Docker images
üéõÔ∏è
Volume
A mechanism for persisting data generated by containers. Volumes exist outside the container lifecycle and can be shared between containers.
Persistent storage for containers
üåê
Port Mapping
Links a port on the host machine to a port inside the container, allowing external access to containerized applications.
Bridge between host and container

Basic Docker Workflow

1
Pull an Image
docker pull nginx
Download a pre-built image from Docker Hub
2
Run a Container
docker run -d -p 80:80 nginx
Create and start a container from the image
3
List Containers
docker ps
View all running containers
4
Execute Commands
docker exec -it <container> bash
Run commands inside a running container
5
Stop Container
docker stop <container>
Gracefully stop a running container
6
Remove Container
docker rm <container>
Delete a stopped container

üìÑ Sample Dockerfile

# Set working directory inside container
WORKDIR /app

COPY package*.json ./
# Install dependencies
RUN npm install
# Copy application code
COPY . .
# Expose port
EXPOSE 3000
# Define startup command
CMD ["npm", "start"]

üí° Pro Tips

Use .dockerignore
Create a .dockerignore file to exclude unnecessary files from your Docker context
Multi-stage Builds
Use multi-stage builds to reduce final image size by separating build and runtime environments
Layer Caching
Order Dockerfile instructions to maximize layer caching - put changing files last
Use Alpine Images
Alpine-based images are smaller and more secure than full distributions
Named Volumes
Use named volumes instead of bind mounts for better portability and management
Health Checks
Add HEALTHCHECK instructions to monitor container status and enable automatic restarts
Environment Variables
Use environment variables for configuration instead of hardcoding values
Regular Cleanup
Run 'docker system prune' regularly to free up disk space from unused resources
