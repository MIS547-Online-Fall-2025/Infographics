Cluster

+
A set of worker machines (nodes) that run containerized applications managed by Kubernetes

Worker Nodes: kubelet, kube-proxy, Container Runtime
Key Features
High availability, automatic scaling, self-healing, load distribution across multiple nodes

Service

Network abstraction layer
+
Provides stable network endpoints for accessing a set of pods, enabling service discovery and load balancing
Service Types
ClusterIP: Internal cluster access only
NodePort: Exposes service on each node's IP
LoadBalancer: Cloud provider load balancer
ExternalName: Maps to external DNS name
YAML Example
apiVersion: v1 kind: Service metadata: name: my-service spec: selector: app: my-app ports: - port: 80 targetPort: 8080 type: ClusterIP
Load balancing Service discovery External access

Namespace

Virtual cluster separation
+
Provides virtual clusters within a physical cluster, enabling resource isolation and multi-tenancy
Default Namespaces
default: Default namespace for objects
kube-system: Kubernetes system components
kube-public: Public resources
kube-node-lease: Node heartbeat information
Benefits
Resource quotas, access control, name scoping, environment separation (dev/staging/prod)
Multi-tenancy Environment separation Resource quotas
