🚀 What is Docker Compose?
Docker Compose is a tool for defining and running multi-container Docker applications. With Compose, you use a YAML file to configure your application's services, networks, and volumes. Then, with a single command, you create and start all the services from your configuration.

Basic Docker Compose Workflow

1
Create docker-compose.yml
touch docker-compose.yml
Define your application services in YAML format
2
Start Services
docker-compose up
Create and start all services defined in compose file
3
View Running Services
docker-compose ps
List all containers managed by compose
4
View Logs
docker-compose logs
Display logs from all services
5
Scale Services
docker-compose up --scale web=3
Scale specific service to multiple instances
6
Stop Services
docker-compose down
Stop and remove containers, networks, and volumes

Essential Docker Compose Commands

🚀
Basic Operations
docker-compose up
Create and start all services
docker-compose up -d
Start services in detached mode
docker-compose down
Stop and remove containers, networks
docker-compose restart
Restart all services
docker-compose stop
Stop services without removing
docker-compose start
Start existing stopped services
🔧
Service Management
docker-compose ps
List running containers
docker-compose ps -a
List all containers (running and stopped)
docker-compose top
Display running processes
docker-compose pause
Pause all services
docker-compose unpause
Unpause all services
docker-compose kill
Force stop services
📊
Monitoring & Logs
docker-compose logs
View output from all services
docker-compose logs -f
Follow log output in real-time
docker-compose logs web
View logs for specific service
docker-compose logs --tail=50
Show last 50 log lines
docker-compose events
Stream container events
docker-compose port web 80
Show public port for service port
🏗️
Building & Images
docker-compose build
Build or rebuild services
docker-compose build --no-cache
Build without using cache
docker-compose up --build
Build images before starting
docker-compose pull
Pull latest images for services
docker-compose push
Push service images to registry
docker-compose images
List images used by services
⚡
Execution & Scaling
docker-compose exec web bash
Execute command in running service
docker-compose run web python manage.py migrate
Run one-time command in new container
docker-compose up --scale web=3
Scale service to 3 instances
docker-compose run --rm web test
Run command and remove container
docker-compose exec -T web cat /etc/hosts
Execute without TTY allocation
docker-compose run --no-deps web test
Run without starting linked services
🧹
Cleanup & Validation
docker-compose down -v
Remove containers, networks, and volumes
docker-compose down --rmi all
Remove containers and all images
docker-compose rm
Remove stopped containers
docker-compose config
Validate and view compose file
docker-compose config --services
List all services
docker-compose config --volumes
List all volumes

💡 Common Use Cases & Examples

Start Development Environment
docker-compose up -d
Start all services in background for development
Rebuild After Code Changes
docker-compose up --build
Rebuild images and start services
Run Database Migrations
docker-compose exec web python manage.py migrate
Execute Django migrations in web container
Scale Web Service
docker-compose up --scale web=5 -d
Run 5 instances of web service
View Application Logs
docker-compose logs -f web
Follow web service logs in real-time
Run Tests
docker-compose run --rm web pytest
Run tests and remove container afterwards
Access Database Console
docker-compose exec db psql -U postgres myapp
Connect to PostgreSQL database
Reset Environment
docker-compose down -v && docker-compose up
Remove volumes and restart fresh
Check Service Health
docker-compose ps
View status of all services
Pull Latest Images
docker-compose pull && docker-compose up -d
Update to latest images and restart
Override for Production
docker-compose -f docker-compose.yml -f docker-compose.prod.yml up
Use production overrides
Backup Database
docker-compose exec db pg_dump -U postgres myapp > backup.sql
Create database backup

💡 Docker Compose Best Practices

Use Environment Files
Store environment variables in .env files instead of hardcoding in compose files
Override Files for Environments
Use docker-compose.override.yml for development and separate files for production
Named Volumes for Data
Use named volumes for persistent data instead of bind mounts in production
Health Checks
Add healthcheck configurations to monitor service status automatically
Depends_on for Startup Order
Use depends_on to control service startup order, but implement proper health checks
Multi-stage Builds
Use multi-stage Dockerfiles to keep production images small and secure
Resource Limits
Set memory and CPU limits to prevent services from consuming all resources
Logging Configuration
Configure logging drivers to manage log file sizes and rotation
Security Best Practices
Don't run containers as root, use secrets for sensitive data, scan images for vulnerabilities
Version Pinning
Pin specific image versions in production to ensure reproducible deployments
Network Isolation
Use custom networks to isolate services and control communication between containers
Regular Cleanup
Regularly run docker system prune to clean up unused containers, networks, and images
